# 개발일지 — KDN 수강생 결과물 GitHub Pages 통합 배포

> 작성: 2026-05 / 대상: `aebonlee` GitHub · 공유 Supabase(`aebonlee's Project`)

## 1. 목표
수강생 결과물(Downloads의 zip 다수)을 선생님 GitHub의 `kdn01 ~ kdn32` 리포로 개별 보관하고,
**GitHub Pages에서 실제 동작/표시**되게 한다. 회사 도메인(dreamitbiz) 미사용. Supabase 연동·로그인 유지.

## 2. 작업 환경 구축
- 이 PC에는 git/gh 미설치 → winget CDN 접속 불가(`0x80072ee7`)로 설치 실패.
- **PortableGit**을 직접 다운로드·압축 해제하여 `C:\Users\Administrator\PortableGit\cmd\git.exe`로 사용.
- git push 시 TLS 인증서 폐기검사 오류(`schannel: CRYPT_E_NO_REVOCATION_CHECK 0x80092012`) 발생
  → `git config --global http.schannelCheckRevoke false`로 해결(openssl 백엔드는 "key too weak"로 실패 → 사용 안 함).
- GitHub API/푸시는 Personal Access Token(`repo` 권한) 사용. 리포 생성·푸시·Pages 활성화·Actions 비활성화 모두 REST API로 자동화.

## 3. 배포 패턴(유형별)
| 유형 | 처리 |
|---|---|
| Vite/React | `base=/kdnNN/`, `outDir=docs` → Pages `main /docs` (`.nojekyll` 필수) |
| CRA(react-scripts) | `homepage=/kdnNN` → `build/` → `docs/`로 복사 |
| Next.js | `output:'export'` + `basePath:'/kdnNN'` + `images.unoptimized` → `out/` → `docs/` (middleware는 export 불가라 제거) |
| 순수 정적 HTML | 루트 그대로 Pages `main /`(루트에 `.nojekyll`) |
| **Streamlit** | **stlite(`@stlite/browser`)** 로 브라우저(WebAssembly/Pyodide) 실행 — `index.html`에 .py를 base64 임베드 |
| Express(ejs) | ejs를 정적 HTML로 렌더링(`defacementType=none`), 자산 경로 상대화 |
| NestJS+Next 풀스택 | 프론트엔드만 정적 export(백엔드 API는 미동작, UI만 표시) |

## 4. 주요 이슈 & 해결
- **워크플로 푸시 거부**: 학생 리포의 `.github/workflows/*.yml`을 토큰 `workflow` 스코프 없이 푸시 불가 → 워크플로 파일 제거 + repo Actions 비활성화(학생 CI가 docs를 덮어쓰지 못하게).
- **Vite base 불일치**: 원래 `/web-test/`, `/kdn-vuln/`, `/ljhKDNnew/` 등 → 전부 `/kdnNN/`로 교정(이게 안 맞으면 JS/CSS 404로 백지).
- **GitHub Pages 첫 빌드 지연**: 이 계정에서 첫 빌드가 매우 느리거나 트리거 안 됨 → 재푸시/빌드 재요청으로 해결.
- **kdn28 vite define**: `process.env` 기반이라 빌드 시 `$env:VITE_*` 설정 필요.
- **PowerShell 5.1 주의**: 네이티브 exe에 `2>&1` 금지(stderr 자동 캡처), 문자열 보간 `"$x:"`는 `"${x}:"`로(드라이브 오인 방지), 한글 SQL은 UTF-8로 읽기/전송.
